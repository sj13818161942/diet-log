# 饮食记录 (diet-log)

## 数据文件

- **食物数据**: `references/food-table.json` (1643条，JSON Lines格式)
- **油和佐料**: `references/oils_seasonings.json`
- **饮食记录**: `{WORKSPACE}/meal_log.json`
- **用户档案**: `{WORKSPACE}/user_profile.json`

> `{WORKSPACE}` = `C:\Users\s\.openclaw\workspace`

## 核心流程

### 第一步：用户档案（首次或缺失时）

若 `user_profile.json` 不存在或不完整，主动询问：

| 字段 | 询问方式 | 说明 |
|------|---------|------|
| 性别 | "请问您的性别是？" | 男/女，影响基础代谢计算 |
| 出生年月 | "请问您的出生年份是？" | 计算年龄 |
| 身高 | "请问您的身高是？" | cm |
| 体重 | "请问您的体重是？" | kg |
| 日常运动 | "您的日常运动量是？" | 久坐/轻度/中度/重度 |
| 健康目标 | "您目前的主要健康目标是？" | 减脂/维持/增肌/其他 |
| 饮食限制 | "有什么饮食限制或过敏吗？" | 素食/忌口/过敏等（可选）|

将档案保存为 `user_profile.json`（见下方格式）。

### 第二步：解析饮食输入

用户输入格式灵活，如：
- "午餐吃了米饭150g，红烧肉100g，青菜200g，用了10g油"
- "早上喝了一杯牛奶，半个苹果"
- "晚餐：一碗面条，两个鸡蛋，番茄炒蛋（用了1勺油）"

**解析步骤：**
1. 识别各食材及其估计份量（重量/个数/杯/碗等）
2. 体积→重量参考：1杯≈200ml，1碗≈250ml，1个鸡蛋≈50g，1勺≈10g，1小把≈30g

### 第三步：查询营养数据

**查询脚本用法：**
```bash
python {WORKSPACE}/skills/diet-log/scripts/query_food.py -d {WORKSPACE}/skills/diet-log/references/food-table.json -o {WORKSPACE}/skills/diet-log/references/oils_seasonings.json -q "食物名" --top-k 5 --json-out
```

> 注意：脚本路径均为绝对路径（基于 `{WORKSPACE}`），不要使用相对路径 `scripts/`。

**匹配策略（三级）：**
1. **精确/包含匹配**：`米饭` → 查找name/nickname包含"米饭"的食物
2. **模糊匹配**：使用相似度算法，找最接近的候选
3. **同类参考**：若某食物完全找不到，查找同类型（如"鸡胸"找不到→用"鸡肉"代替）

**油和佐料处理（重要 — 必须纳入计算）：**
- 若用户提到具体油类（花生油/大豆油等），查 `oils_seasonings.json`
- **用户未提及油时**，根据餐食类型自动估算（因为几乎所有中式烹调都用到油）：
  - 面条/米粉类：**默认加 10g 植物油** + 酱油 5g + 盐 1g
  - 炒菜/米饭套餐：**默认加 10g 植物油**
  - 蒸/煮/汤：**默认加 5g 植物油**
  - 用户明确说"少油/无油"时，按用户要求
- 佐料中的酱油/盐/味精含大量钠，每餐应注明钠摄入量
- **每次分析时，主动在输出中说明油和调料的估算来源**（"🌱 大豆油 10g（默认估算）"）

**无法识别的食物：** 向用户提问确认，如"请问您说的【xxx】具体是什么食物？大概吃了多少？"

### 第四步：计算并输出营养分析

**计算各食材营养之和**，按以下分类汇总输出：

**热量与宏量营养素：**
- 热量（kcal）
- 蛋白质（g）
- 脂肪（g）
- 碳水化合物（g）
- 膳食纤维（g）

**脂肪酸：**
- 单不饱和脂肪酸（g）
- 多不饱和脂肪酸（g）
- 反式脂肪酸（g）
- 胆固醇（mg）

**矿物质：**
- 钙/磷/钾/钠/镁/铁/锌/铜/锰/硒（mg）

**维生素：**
- 维生素A/维生素C/维生素D/维生素E/维生素K
- 维生素B1/维生素B2/维生素B3/维生素B6/维生素B12
- 叶酸（B9）

**输出格式：**
```
🍽️ 【午餐营养分析】2026-04-15 12:30

食物构成：
  · 米饭 150g → 热量 174kcal | 蛋白质 3.9g | 脂肪 0.5g | 碳水 38.9g
  · 红烧肉 100g → 热量 215kcal | 蛋白质 14g | 脂肪 17g | 碳水 2g
  · 青菜 200g → 热量 34kcal | 蛋白质 2.4g | 脂肪 0.2g | 碳水 3.4g
  · 花生油 10g → 热量 90kcal | 脂肪 10g

━━━━━━━━━━━━━━━━━━━━
合计：热量 513kcal | 蛋白质 20.3g | 脂肪 27.7g | 碳水 44.3g | 纤维 3.2g
钠 480mg | 铁 2.8mg | 锌 3.1mg | 维生素C 45mg | 维生素E 3.2mg
```

### 第五步：保存记录

将本餐记录追加到 `meal_log.json`（JSON Lines格式，每行一条记录）：

```json
{"timestamp":"2026-04-15T12:30:00+08:00","meal_type":"lunch","foods":[{"name":"米饭","amount":"150g","energy_kcal":174,"protein_g":3.9,"fat_g":0.5,"carbs_g":38.9}],"total":{"energy_kcal":513,"protein_g":20.3,"fat_g":27.7,"carbs_g":44.3},"user_profile_summary":"..."}
```

> **重要：** 每次记录后必须立即持久化到磁盘，不依赖任何中间缓存。

## 阶段性统计

当用户要求统计（如"最近7天的饮食情况"）：

**统计脚本：** 读取 `meal_log.json`，按日期/周/月聚合：

```bash
python {WORKSPACE}/skills/diet-log/scripts/stats_meal.py -d {WORKSPACE}/meal_log.json --days 7 --profile {WORKSPACE}/user_profile.json
```

**统计内容：**
- 日均/周均总热量及趋势
- 三大营养素比例（蛋白质/脂肪/碳水）是否符合目标
- 平均每日钠摄入量
- 各营养素日均摄入量 vs 推荐值（基于用户档案中的年龄/性别/活动水平）

**输出格式：**
```
📊 【最近7天饮食统计】2026-04-08 ~ 2026-04-15

总热量：13,245 kcal（日均 1,892 kcal）
目标缺口/超额：约 -108 kcal/天（维持体重目标下，偏低）

营养素比例：
  蛋白质：日均 68g（占14%）✅  建议15-20%
  脂肪：  日均 56g（占27%）⚠️  建议20-30%
  碳水：  日均 268g（占59%）✅  建议45-65%

矿物质（日均）：
  钠 3,200mg（⚠️ 超出建议上限5,000mg/天，偏高）
  钙 420mg（⚠️ 建议800mg/天，偏低）
  铁 13mg（✅ 正常）

建议：
  · 脂肪摄入比例略高，建议减少烹调油用量或选择更瘦的肉类
  · 钙摄入偏低，建议每天喝一杯牛奶或多吃豆制品
```

## 编码说明

`food-table.json` 中的营养值格式需解析：
- `"37千卡"` → 提取 `37.0` kcal
- `"1.5克"` → 提取 `1.5` g
- `"毫克"` / `"克"` 单独出现 → 视作 `0`
- 脂肪酸/氨基酸等缺失同理

使用 `scripts/query_food.py` 的 `--json-out` 模式可自动完成数值解析。

## 数据完整性保障

1. **每次记录后立即写入磁盘**（追加到 meal_log.json）
2. **meal_log.json 为 JSON Lines 格式**，每行一条记录，即使文件损坏也可逐行恢复
3. **user_profile.json 独立存储**，与饮食记录分离
4. **原始数据文件（food-table.json、oils_seasonings.json）在 skill 内**，不依赖外部网络
